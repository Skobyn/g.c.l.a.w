   the active phase this iteration; don't jump ahead.
3. TDD loop inside each phase:
     a. Write failing tests that express the acceptance criteria.
     b. Run the tests — confirm RED.
     c. Write the minimum production code to turn them GREEN.
     d. Refactor without breaking tests.
4. Acceptance: a phase is `[x]` only when EVERY item under its "C — Completion"
   section is verified (tests green, lint clean, file exists, route responds).

   e.g. `feat(tools): add ToolCatalog data model and Firestore repo`). DO NOT
   push unless the user explicitly asks.

   pattern; every new kind (mcp, http_api, code_exec) ships with its own
   nested toggle so a broken integration can't brick the built-in tools.
7. If blocked (missing credentials, ambiguous design, destructive op needed),
   flip the phase to `[!]` with a one-line reason, surface to the user, stop.
8. Keep scope tight. No speculative refactors. No "while we're here" cleanup.
9. After a phase is verified, flip to `[x]` and add a one-line delta note
   under the phase (e.g. `DONE 2026-04-18: wired ToolRepo + CRUD routes...`).
10. When Phase 7 is `[x]`, print the final summary and stop the loop (no
    ScheduleWakeup).

================================================================================
GLOBAL DESIGN NOTES
================================================================================
- Catalog shape mirrors the existing ModelCatalog (providers/models under
  config/catalog/). Tools live at config/tools/{id}. Every record carries
  {id, name, kind, enabled, config: <kind-specific union>, created_at,
   updated_at, credential_ref: str | None}.
- Four kinds:
    * `builtin`   — pointer to one of src/gclaw/tools/*.py functions; config
                    is just the function's dotted path.
    * `mcp`       — external MCP server; config has {transport, endpoint,
                    allowed_tools: list[str] | None, env: dict[str, str]}.
    * `http_api`  — wraps an OpenAPI/Swagger spec. Config has {spec_url OR
                    spec_inline, base_url, auth: AuthSpec, allowed_operations}.
                    Internally materialized as an in-process MCP server so
                    agents see a single tool-binding abstraction.
    * `code_exec` — sandboxed Python/bash runner. Config has {runtime,
                    timeout_seconds, memory_mb, network: 'none'|'egress-only',
                    allowed_modules: list[str]}.
- Credentials go in Secret Manager via the existing SM service; Tool records
  hold the SM resource path only.
- Per-agent binding: extend AgentOverride.tools with
  `catalog_tool_ids: list[str]`. Agents grant catalog tools via this list;
  legacy allow/deny lists still govern builtin function names for back-compat.
- ToolBindingService is the new public surface AgentFactory calls to resolve
  an agent's list of ADK-callable tools from (builtin_tools + catalog_entries).
- OpenAPI→MCP wrap lives in src/gclaw/tools/openapi_mcp/, emits one MCP tool
  per operation; handles apiKey(header/query), http(bearer/basic),
  oauth2(bearer from SM).
- Code-exec sandbox runs as a Cloud Run sibling service (`gclaw-sandbox`);
  calls happen over signed-JWT HTTP with a 30s hard timeout. Local dev uses
  subprocess+resource-limits fallback.
- Toggle names (Settings):
    TOOL_CATALOG_ENABLED     (default true)
    TOOL_MCP_ENABLED         (default true)
    TOOL_HTTP_ENABLED        (default true)
    TOOL_CODE_EXEC_ENABLED   (default FALSE — opt-in; infra dependency)
- Test strategy: every phase starts with failing tests; integration tests
  stub Firestore (repos accept an injected client) and stub the MCP/httpx
  transports so CI stays offline.

PHASE 1 — Data model + Firestore repo + service-layer CRUD
================================================================================
S — Specification
  Goal: Persist a ToolRecord in Firestore with a discriminated-union config
  field, through a ToolRepo + ToolCatalogService that mirror the ModelCatalog
  surface. No admin-facing routes yet; no agent binding yet.
  Acceptance:
    - Create / read / list / update / delete round-trip cleanly through an
      in-memory fake Firestore.
    - Discriminated-union config validation: wrong keys for kind →
      pydantic.ValidationError.
    - Enabled/disabled flag respected by list_enabled().
    - Public module surface documented in src/gclaw/tools/catalog/__init__.py.

P — Pseudocode
  1. src/gclaw/tools/catalog/models.py:
       class ToolKind(StrEnum): BUILTIN, MCP, HTTP_API, CODE_EXEC
       class BuiltinConfig(BaseModel):  function_path: str
       class McpConfig(BaseModel):
           transport: Literal["stdio","sse","http"]
           endpoint: str
           allowed_tools: list[str] | None = None
           env: dict[str,str] = {}
       class HttpApiConfig(BaseModel):
           spec_url: str | None = None
           spec_inline: dict | None = None
           base_url: str
           auth: AuthSpec
           allowed_operations: list[str] | None = None
       class CodeExecConfig(BaseModel):
           runtime: Literal["python3.12","bash"]
           timeout_seconds: int = 30
           memory_mb: int = 256
           network: Literal["none","egress-only"] = "none"
           allowed_modules: list[str] = []
       ToolConfig = Annotated[Union[...], Discriminator("kind")]
       class ToolRecord(BaseModel):
           id: str, name: str, kind: ToolKind, enabled: bool = True,
           config: ToolConfig, credential_ref: str | None = None,
           created_at: datetime, updated_at: datetime
  2. src/gclaw/firestore/tool_repo.py: ToolRepo.create/get/list/update/delete.
  3. src/gclaw/tools/catalog/service.py: ToolCatalogService wrapping ToolRepo +

PHASE 6 — Code-exec-kind (sandboxed runner)
================================================================================
S — Specification
  Goal: Safe code execution as a tool. Agents submit Python/bash; a
  sandboxed runner executes with resource limits; stdout/stderr/exit_code
  returned. Default disabled because infra setup is non-trivial.
  Acceptance:
    - Local dev path: subprocess with ulimit (CPU, memory, no network).
    - Prod path: Cloud Run sibling service `gclaw-sandbox` reached via
      signed JWT; adapter hits POST /execute.
    - Policy enforcement: memory cap, timeout, no network egress (unless
      network='egress-only'), allowed_modules whitelist for Python.
    - Tool output: {stdout, stderr, exit_code, duration_ms, truncated}.
